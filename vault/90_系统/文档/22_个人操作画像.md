# 个人操作画像

个人操作画像是 kos 中用于长期个性化协作的高阶个人上下文对象。它不回答“我是谁”，而回答：

```text
在当前阶段，哪些关于我的工作方式、认知偏好、协作方式和行动模式的假设，已经足够可靠，可以帮助 kos 和 Agent 更好地协作？
```

## 为什么不是人格画像

Agent 在交互中学到的是用户表现：提需求方式、纠错方式、输出偏好、项目类型和协作节奏。这些是行为痕迹，不等于真实本我。

盖洛普、荣格、MBTI、Big Five 等测评也不是最终答案。它们只是结构化自述或测量结果，可以提供假设，但不能单独拥有解释权。

因此 kos 使用“个人操作画像”这个名称，强调它是操作性上下文，不是人格诊断。

## 证据来源

个人操作画像可以引用多种证据：

| 来源 | 作用 | 限制 |
|---|---|---|
| 测评结果 | 提供结构化假设 | 不能直接当成结论 |
| Agent 交互观察 | 反映协作表现 | 只能说明交互中的用户表现 |
| 日记和复盘 | 反映真实生活中的自我观察 | 需要长期重复出现才稳定 |
| 项目行为 | 反映行动模式 | 需要结合情境解释 |
| 他人反馈 | 提供外部观察 | 仍需用户确认 |

任何单一来源都不能直接写成 active profile。

## 对象位置

```text
25_个人操作画像/
```

对象类型：

```yaml
type: personal_operating_profile
status: draft
confidence: draft
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: []
related_reflections: []
related_methods: []
related_projects: []
applies_to_skills: []
reviewed: false
tags: []
```

## 生命周期

```text
draft -> reviewed -> active -> archived
```

规则：

- Agent 可以创建 `draft`。
- `reviewed` 必须由用户确认。
- `active` 表示当前可用于调整协作方式。
- `archived` 表示旧画像不再作为当前协作依据。

`confidence` 使用：

```text
draft -> verified -> mature
```

`verified` 和 `mature` 也必须有人确认。

## 正文结构

推荐结构：

```text
## 当前可用结论
## 支持证据
## 适用场景
## 不适用场景
## 协作偏好
## 高能量任务
## 低能量任务
## 决策盲区
## Agent 应如何使用
## 仍需验证的假设
## 已被推翻的旧判断
## 版本记录
```

其中最重要的是“不适用场景”“仍需验证的假设”和“已被推翻的旧判断”。它们防止画像变成静态标签。

## 创建画像

使用 Skill：

```text
/kos-update-personal-profile <画像主题或更新说明>
```

也可以直接调用 Harness：

```bash
kos-harness create --kind personal_operating_profile --title "个人操作画像" \
  --source "盖洛普优势结果" \
  --related-reflection "近期项目复盘" \
  --conclusion "当前可用结论" \
  --evidence "支持证据" \
  --collaboration-preference "协作偏好"
```

预览：

```bash
kos-harness create --kind personal_operating_profile --title "个人操作画像" --dry-run
```

## Skill 如何使用画像

Skill 可以在用户明确要求时读取个人操作画像，例如：

```text
根据我的个人操作画像，帮我设计这个项目推进方式。
```

但 Skill 不应默认让画像污染所有输出。画像只能作为协作假设使用，不能作为对用户的固定标签。

正确表达：

```text
根据当前画像中的协作假设，你可能更适合...
```

错误表达：

```text
你就是这种人格，所以你应该...
```

## 检查

```bash
kos-harness validate
```

如果画像进入 `reviewed`、`active`、`verified` 或 `mature`，必须有人工确认元数据，例如：

```yaml
human_confirmed: true
```
