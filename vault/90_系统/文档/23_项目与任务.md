# 项目与任务

kos 的项目管理不是把所有事情都变成待办清单。项目负责保存目标、背景、决策、进展和沉淀；任务负责指出下一步行动。

核心原则：

```text
Project 记录为什么做、做到什么程度、如何判断进展。
Task 记录下一步做什么。
Dashboard 帮你每天看见该推进什么。
```

## 项目是什么

Project 是持续行动容器，位于：

```text
30_项目/
```

Project 适合放：

- 学习计划
- 研究方向
- 写作计划
- 产品或代码项目
- 系统建设事项
- 需要多次推进和复盘的长期行动

临时想法、单次任务、摘要和研究报告不应直接写成 Project。

## 项目字段

Project frontmatter：

```yaml
type: project
title: "项目名"
status: idea
category: other
priority: P2
area: "[[领域]]"
goal: ""
current_stage: ""
due: ""
created: YYYY-MM-DD
updated: YYYY-MM-DD
related_sources: []
related_research: []
related_concepts: []
related_methods: []
tags: []
```

`status` 可用：

```text
idea / active / paused / blocked / completed / archived / cancelled
```

`category` 可用：

```text
learning / research / writing / product / coding / investment / career / system / other
```

`priority` 可用：

```text
P0 / P1 / P2 / P3 / P4
```

`completed`、`archived`、`cancelled` 是受保护状态，必须由用户明确确认。

## 项目正文

Project 正文应包含：

```text
## 背景
### 为什么做
### 项目目标
### 当前阶段
### 当前问题
### 成功指标
### 限制条件

## 决策日志
## 进展
## 当前任务
## 阶段性复盘
## 最终成果
## 最终沉淀
## 相关
```

项目页记录长期上下文。不要把它只当成任务列表。

## 项目启动

如果项目还没想清楚，先用 kickoff：

```text
/kos-kickoff <项目想法或收件箱文件路径>
```

它默认只创建启动计划：

```text
90_系统/工作流/项目启动计划/
```

启动计划用于澄清：

- 为什么做
- 项目目标
- 不做什么
- 阶段
- 成功指标
- 初始任务
- 限制和风险
- 需要用户确认的问题

未经用户确认时，不应直接创建 Project。

## 创建项目

确认要纳入项目管理后使用：

```text
/kos-create-project <项目想法或项目说明>
```

或调用 Harness：

```bash
python3 90_系统/harness/create_project.py "项目名" \
  --status idea \
  --category other \
  --priority P2 \
  --goal "项目目标" \
  --why "为什么做" \
  --current-stage "想法澄清阶段" \
  --success "成功指标" \
  --task "下一步行动"
```

默认状态应是 `idea`。只有用户明确说“开始执行”“设为 active”时，才使用 `active`。

## 更新项目

推进项目时使用：

```text
/kos-update-project <项目名或路径> <进展、任务、决策或复盘>
```

或调用 Harness：

```bash
python3 90_系统/harness/update_project.py "项目名" \
  --progress "完成了某项推进" \
  --task "下一步行动" \
  --decision "记录一个关键决策" \
  --review "阶段性复盘"
```

状态变更到受保护状态时必须显式确认：

```bash
python3 90_系统/harness/update_project.py "项目名" --status completed --human-confirmed
```

更新 Project 会更新 `updated` 日期，并把内容追加到对应章节。

## 任务是什么

Task 是具体行动项，位于：

```text
31_任务/
```

也可以直接写在 Project 的：

```text
## 当前任务
```

当前实现中，任务有两种形态：

1. 项目页内 checklist：适合多数项目推进。
2. 独立 `type: task` 文件：适合跨项目、需要单独跟踪或出现在工作台的行动项。

独立 Task frontmatter：

```yaml
type: task
title: "任务名"
status: todo
project: "[[30_项目/某项目]]"
priority: P2
due: ""
created: YYYY-MM-DD
completed: ""
tags: []
```

`status` 可用：

```text
todo / doing / done / blocked / cancelled
```

注意：当前 framework 对 Project 有完整 schema；Task 目前主要由对象规范和模板约束，schema 校验仍是轻量状态。

## 今日工作台如何使用项目和任务

每天开始时运行：

```text
/kos-start-my-day
```

或：

```bash
python3 90_系统/harness/generate_daily_dashboard.py
```

它会生成或更新：

```text
00_工作台/今日工作台.md
```

工作台会聚合：

- active / idea / blocked / paused 项目
- 太久未更新的 active 项目
- 项目页 `## 当前任务` 中的未完成 checklist
- `31_任务/` 下的 todo / doing / blocked 任务
- 待处理输入源和待审核 AI 产物

工作台可以建议今日主线，但不能替用户确认今日主线。

## 推荐工作流

```text
想法或材料
-> /kos-kickoff 生成启动计划
-> 用户确认
-> /kos-create-project 创建 Project
-> /kos-start-my-day 每天看当前项目和任务
-> /kos-update-project 记录推进、决策、复盘和状态变化
-> 阶段性复盘后沉淀 Method / Concept / Research
```

## 常见错误

- 把临时想法直接设为 active Project。
- 跳过 kickoff，导致项目目标和成功指标不清楚。
- 在 Project 里只堆任务，不记录为什么做和决策依据。
- 把任务、摘要或研究报告误放进 `30_项目/`。
- 未经确认就把 Project 标记为 completed、archived 或 cancelled。
- 用 Agent 自动决定今日主线。
- 删除旧决策日志和进展记录，而不是追加新记录。

## 检查

```bash
python3 90_系统/harness/validate_paths.py --format markdown
python3 90_系统/harness/validate_schema.py --format markdown
python3 90_系统/harness/validate_state.py --format markdown
python3 90_系统/harness/validate_permissions.py --format markdown
python3 90_系统/harness/generate_health_report.py
```

如果 Project schema、状态或权限检查失败，先修项目结构，再继续推进任务。
