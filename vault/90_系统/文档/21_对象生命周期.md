# 对象生命周期

kos 的核心原则是：AI 可以创建草稿和待审核对象，人负责最终判断。

对象字段和状态的完整规则见：

```text
90_系统/规则/对象规范.md
90_系统/文档/20_对象模型与模板.md
```

## Source

```text
captured -> extracted -> summarized -> reviewed -> linked -> archived
```

## Research

```text
draft -> reviewed -> complete -> archived
```

`reviewed` 和 `complete` 需要人工确认。

## Concept

```text
draft -> verified -> mature
```

AI 只能创建 `draft`。

## Method

```text
candidate -> usable -> trusted -> deprecated
```

`usable` 和 `trusted` 需要实践验证和人工确认。

## Skill

```text
incubator -> core / integrations / personal -> archived
```

正式晋升前必须通过 eval，并由用户确认。

## Personal Operating Profile

```text
draft -> reviewed -> active -> archived
```

AI 可以创建 `draft`，但不能把画像标记为 `reviewed` 或 `active`。画像只能作为协作假设使用，不是人格诊断。

## 检查状态

```bash
kos-harness validate
```

如果检查失败，先修正对象状态和人工确认边界，再继续让 Skill 处理对象。
