# 快速开始

## 1. 检查系统

在 vault 根目录运行：

```bash
python3 90_系统/harness/generate_health_report.py
```

如果使用 Hermes，执行：

```text
/kos-system-check
```

如果使用 Codex 或 Claude Code，先让 agent 读取：

```text
.kos.md
AGENTS.md 或 CLAUDE.md
90_系统/规则/对象规范.md
```

## 2. 开始一天

```text
/kos-start-my-day
```

这会生成或更新：

```text
00_工作台/今日工作台.md
```

## 3. 收集输入源

```text
/kos-ingest <文章、书籍、论文、视频、播客或报告>
```

输入源默认进入：

```text
11_原材料/
```

## 4. 处理输入源

```text
/kos-extract <Source>
/kos-summarize <Source>
```

## 5. 形成知识

```text
/kos-research <研究问题>
/kos-create-concept <概念>
```

## 6. 推进项目

```text
/kos-kickoff <项目想法>
/kos-create-project <项目说明>
/kos-update-project <项目进展>
```

## 7. 建立个人操作画像

```text
/kos-update-personal-profile <测评结果、复盘或协作偏好>
```

这会创建 `25_个人操作画像/` 下的 draft。它只表示当前可验证的协作假设，不是人格诊断。

## 8. 下一步阅读

- `10_目录结构.md`：理解 kos 预置目录和动态目录。
- `20_对象模型与模板.md`：理解对象、frontmatter、模板和 schema。
- `22_个人操作画像.md`：理解个性化协作假设如何进入 kos。
- `23_项目与任务.md`：理解项目启动、项目更新、任务和今日工作台。
- `24_读书与阅读.md`：理解书籍 Source、摘录、摘要、研究和概念沉淀。
- `30_Harness与系统检查.md`：理解健康检查和确定性脚本。
- `40_Skill管理与防腐.md`：理解 Skill scope、晋升和人工确认。
- `41_Skill Eval与防腐.md`：理解如何用 eval 防止 Skill 行为漂移。
- `50_扩展与个人化.md`：理解 integration、personal 和 incubator 的边界。
- `60_Framework同步.md`：理解如何从 framework 更新个人 vault。
- `70_Agent后端.md`：理解 Hermes、Codex、Claude Code 如何作为不同后端操作同一个 vault。
- `90_故障排查.md`：检查失败时按顺序定位问题。
