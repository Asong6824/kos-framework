# 快速开始

## 1. 启动 kos-agent

把 kos Companion 发布目录放到：

```text
<Vault>/.obsidian/plugins/kos-companion
```

安装或覆盖更新后，在 Obsidian 中启用或重新加载 kos Companion。桌面端 kos-agent 需要 Node.js 22.19+；看板和 Reader 本身不需要模型调用。

打开右侧 kos Agent。首次使用点击“配置模型”，填写 provider、model ID、API key；使用自定义中转时同时填写 Base URL 和 API 协议。

API key 由 kos-agent 保存在自身配置目录，不进入 Vault 笔记或 Obsidian 插件数据。

kos-agent 只有 YOLO 模式。启用并使用 Agent，表示它可以在宿主进程已有权限范围内读写文件和执行命令，不会为每次工具调用弹出授权框。你可以查看工具调用和 diff、随时停止运行；涉及目标、事实判断或人工审阅结论时，Agent 应通过提问等待你的决定。

## 2. 打开看板

从左侧功能区或命令面板打开 kos 看板。“今日、行动、输入、知识、审阅与复盘、系统”六个区块位于同一个纵向页面，不需要切换模块。打开、滚动、筛选或刷新看板不会自动调用 Agent；只有点击“开始一天”“结束一天”或其他明确 Agent 动作时才会提交工作流。

## 3. 在 Obsidian 中阅读 PDF 或 EPUB

把 PDF 或无 DRM 的 EPUB 放进 Vault 后：

- EPUB 会出现在 Obsidian 文件树，点击文件即可用 kos Reader 打开。
- PDF 仍由 Obsidian 原生 PDF 视图打开；需要 kos Reader 时，在文件上右键选择“使用 kos Reader 打开”。
- 也可以选中文件后运行命令“使用 kos Reader 打开当前文件”。
- 如果文件尚未关联 Source，插件会按书籍 Source 模板自动创建关联记录，不调用模型。

Reader 是独立中央视图，不嵌入看板。当前支持 Source Markdown、本地有文字层 PDF、无 DRM EPUB、目录、翻页、全文搜索和进度恢复；PDF 默认纵向连续滚动，EPUB 默认跨章节连续滚动并可从工具栏切换分页阅读。选中文本后可创建持久划线或彩色批注，内容写入关联 Extract，重新打开后仍会回显，并可从批注面板跳回或删除。选区、当前页/章节和本次阅读会话 Summary 都只预填到 Agent 输入框，不会自动发送。

## 4. 检查系统

在 kos Agent 侧栏点击系统检查按钮，或在 Vault 根目录通过插件随包脚本运行无模型检查：

```bash
node .obsidian/plugins/kos-companion/kos-agent/dist/kos-harness.mjs validate
```

## 5. 开始一天

```text
帮我开始今天的工作：检查待审核、进行中项目和任务，更新今日工作台，并告诉我最值得推进的三件事。
```

## 6. 收集和处理输入

```text
把这份材料作为输入源收集到 kos，并提取重点、形成摘要：<链接或内容>
```

输入源默认进入 `11_原材料/`。Agent 会按需继续形成摘录、摘要、研究和概念。

## 7. 形成知识

```text
围绕 <研究问题> 做研究，把结论、证据和不确定性写入 kos；必要时创建相关概念。
```

## 8. 推进项目

```text
把 <项目想法> 建成项目，明确目标、当前阶段和下一步任务，然后开始推进。
```

## 9. 建立个人操作画像

```text
根据这些测评结果、复盘或协作偏好更新我的个人操作画像：<内容>
```

这会创建 `42_个人操作画像/` 下的 draft。它只表示当前可验证的协作假设，不是人格诊断。
