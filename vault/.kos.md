# kos Vault Context

This directory is a kos vault.

kos is a Markdown-vault-based personal knowledge operating system. It organizes inputs, extracts, summaries, research, concepts, projects, tasks, diaries, reflections, methods, signals, and agent skills as checkable Markdown objects.

## Runtime Model

- Markdown files are the source of truth.
- Obsidian or any Markdown editor can be used as the visual front end.
- kos-agent is the official runtime and is strongly bound to this Vault.
- The kos-agent Harness owns model access, the agent loop, tools, Skills, sessions, deterministic operations, validation, and feedback.
- kos-agent runs in YOLO mode and may create files, execute commands, and perform any legal object-state transition without per-operation approval.
- `ask_question` is available for missing information or deliberate review points, but it is not a permission gate.

## Required Reading For Agents

Before creating or modifying objects, read:

- `90_系统/规则/对象规范.md`
- the relevant template under `90_系统/模板/`
- the relevant user doc under `90_系统/文档/`

## Root Rules

- Treat the directory containing this `.kos.md` file as the vault root.
- Write paths relative to the vault root.
- Do not create a nested vault inside this vault.
- Do not overwrite user-authored sections without reading them first.
- Prefer kos-agent deterministic operations and validation feedback for object creation and state changes.
