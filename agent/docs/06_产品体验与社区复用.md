# 产品体验与社区复用

## 1. 体验参照

kos-agent 的交互不是传统聊天机器人，也不重新发明一套知识库对话范式。主要参照：

- Claude Code：目标驱动、连续工具执行、流式反馈、session、steering、compaction 和低摩擦工作方式。
- Claudian：把 coding-agent 体验嵌入 Obsidian，包括 sidebar chat、上下文引用、工具卡片、diff、inline edit 和会话管理。

“接近”指用户的操作预期和信息结构熟悉，不要求像素级复制，也不要求支持两者全部功能。

## 2. kos-agent 交互基线

### 2.1 对话主界面

- 同一个 kos Obsidian 插件新增独立 Agent sidebar/view。
- 支持流式回答、thinking block 和 Markdown 渲染。
- 工具调用按卡片展示，默认折叠冗长输出。
- 文件路径可点击并在 Obsidian 中打开。
- `edit`/`write` 展示清晰 diff。
- 随时可以 Stop；运行中可以发送 steering 和 follow-up。
- 不显示 plan mode、permission mode 或 approval 控件。

### 2.2 Context 输入

- `@` 引用 Vault 文件、目录和 kos 对象。
- 一键附加当前文件、当前选区和看板选中对象。
- 支持拖入或粘贴图片，沿用 Pi attachment 能力。
- `/` 展示命令、prompt template 和显式 Skill。
- 自动触发 Skill 时，在消息流中显示激活的 Skill 和原因。
- Context chips 可以在发送前移除，发送后可以追踪实际注入内容。

### 2.3 Session

- 新建、命名、恢复和搜索会话。
- 多 tab 并行保存不同主题。
- fork/tree/compact 沿用 Pi session 语义。
- 会话显示关联 Project、Research 或当前对象，但不强制一对一。
- Session 不是知识真相源；长期结论仍需沉淀为 Markdown 对象。

### 2.4 Obsidian 原生操作

- 从看板、待审核项、文件菜单和编辑器选区发起 Agent 对话。
- Inline edit 使用原生选区和 word-level diff，交互参考 Claudian。
- Agent 写入后由现有 metadataCache/KosIndex 事件自动刷新看板。
- 看板直接操作和 Agent 写入共享同一对象规范与 Validator。

### 2.5 状态与费用

- 显示当前 provider、model 和 thinking level。
- 显示 context usage、tokens、cache、费用和运行耗时。
- 明确区分运行中、等待用户回答、已停止、失败和完成。
- Provider、host 或版本异常时给出可操作错误，不只显示自由文本堆栈。

## 3. 与参照产品的取舍

| 能力 | Claude Code | Claudian | kos-agent 决策 |
|---|---|---|---|
| 连续工具执行 | 核心体验 | 内嵌展示 | 保留，固定 YOLO |
| Plan mode | 可用 | UI 可切换 | 不实现 |
| Permission mode | 多种策略 | 有 approval UI | 不实现 |
| `@mention` | 文件引用 | Vault/外部 context | 保留并做 kos 对象增强 |
| Slash commands/Skills | 支持 | 支持 | 继承 Pi 并加入 kos scope |
| Session resume/fork/compact | 支持 | 支持 | 继承 Pi |
| Tool cards/diff | 支持 | Obsidian 原生展示 | 按 Claudian 体验实现 |
| Inline edit | IDE/terminal 工作流 | Obsidian 原生 | 选择性移植 Claudian 实现及测试 |
| Multi-provider adapter | Claude 产品 | Claude/Codex/OpenCode/Pi | kos 只提供 kos-agent，模型 provider 可切换 |
| 看板和对象模型 | 无 | 无 | kos 专有能力 |

## 4. Claudian 复用策略

Claudian 采用 MIT License，已经实现大量与 kos 目标相同的 Obsidian 交互：

- Conversation repository、session storage 和 resume dropdown。
- Chat view、conversation controller 和 tab session。
- Tool call、diff、AskUserQuestion 渲染。
- File/Image context tray、Vault mention cache 和 dropdown。
- Slash command、model picker 和 inline edit。Claudian 的 MCP settings 不移植。
- Pi RPC provider adapter 与 tool normalization。

确定采用选择性移植：移植 chat、session、context、tool、diff、question 和 inline-edit 模块及对应测试，适配现有 kos 插件样式与数据层。不移植 Claudian 的多 Agent provider registry、MCP settings 和 permission/plan mode。

## 5. 社区调研要求

每个缺失能力在实现前建立候选清单，至少比较：

- 功能覆盖和与 Pi 的集成成本。
- License、copyright 和分发义务。
- 最近维护、release 频率、issue 状态和 bus factor。
- 自动化测试和真实使用量。
- 网络请求、密钥、文件、命令和进程权限。
- 输出 schema、context 成本和错误质量。
- 跨平台、Obsidian/Electron 和 Node 版本兼容性。
- 锁版本、升级和退出方案。

调研结论放在 `agent/docs/decisions/` 或对应开发文档中。没有候选满足要求时，记录拒绝原因后才能自研。
