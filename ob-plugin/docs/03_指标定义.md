# kos Obsidian 插件指标定义

> 版本：v0.1 草案
> 本文档是全部量化展示的唯一口径标准。`src/core/metrics.ts` 的函数与本文条目一一对应（函数名后标注指标编号），测试用例必须覆盖每条目列出的边界情况。
> 数据契约：`vault/90_系统/规则/对象规范.md`

## 0. 通用约定

1. **日期口径**：所有"日"指设备本地时区的日历日，统一用 `YYYY-MM-DD` 字符串比较。`created`/`completed`/`date`/`due` 只取前 10 位。
2. **权威时间**：只认 frontmatter 字段（`created`、`completed`、`date`），不使用文件 ctime/mtime。字段缺失时该对象不参与时间类指标，并计入"字段缺失"提示。
3. **两种累计口径**，全文档严格区分：
   - **存活口径**：当前存在的有效对象数，可增可减（排除 archived / cancelled / deprecated / ignored）。
   - **累计口径**：历史上创建过的总量，只增不减（含已归档），用于热力图、复利曲线、里程碑。
4. **streak 端点**：当天尚无活动时，streak 不死，以昨天为最新连续端点；当天一旦产生活动即续上。
5. **补记规则**：日记按其 frontmatter `date` 计入对应日期，与文件何时创建无关。
6. **补落快照**：插件未运行的日期，总数类快照无法精确还原（对象可能已被删除或流转），补落快照标记 `estimated: true`，仅用于曲线展示，不参与环比判定。
7. **快照差分**：source 无 `updated` 字段，状态流转时间不可知，凡涉及"某日处理了多少"的口径一律用相邻两日快照差分（见 M4）。

## 1. 资产与增长

### M1 知识资产总数（存活口径）
- 公式：`concept 全状态 + method(排除 deprecated) + research(排除 archived)`。
- 展示：驾驶舱顶部大数字 + 分类构成。
- 边界：concept 无 archived 状态，全计入；方法 deprecated 后不计。

### M2 本周 / 本月新增（累计口径）
- 公式：`created ∈ [本周一, 今天]`（周起始日可配，默认周一）/ `created ∈ [本月 1 日, 今天]` 的对象数，按类型分列。
- 环比：与上一等长区间对比，展示 `+N (+X%)`；上期为 0 时只显示绝对值，不显示百分比。

### M3 知识复利曲线（累计口径）
- 数据：`concept + method + research` 按 `created` 的日累计和，叠加 `source` 累计作为对照线。
- 展示：自最早 `created` 至今的折线；纵轴总量，横轴日期；标注 M2 的周环比。
- 边界：安装前历史由 `created` 回填，不需要快照。

## 2. 日常活跃

### M4 今日进度环
- 两个环，数据源互不混用：
  - **任务环**：分子 = 今日 `completed` 的 task 数；分母 = 分子 + `due ≤ 今日` 且 status ∈ {todo, doing} 的 task 数。分母为 0 时不显示比率，只显示完成数。
  - **输入环**：今日处理的 source 数（快照差分：昨日快照中 in-flight 减少量 + 今日 reviewed/linked 增加量）；目标值 = 昨日快照的积压数，为 0 时显示"无积压"。
- 边界：项目页内 checkbox 无完成时间，只计入 M11 项目推进度，不计入任务环。

### M5 活动热力图
- 日分值：`dayScore = 当日创建对象数(全部类型, created=当日) + 当日完成任务数(completed=当日) + 日记存在(0/1, 可配关闭)`。
- 分档：0 / 1–2 / 3–5 / 6–9 / ≥10，五档色阶。
- 数据：`created`/`completed`/日记 `date` 回填全量历史，安装第一天即完整。
- 边界：ignored source 的创建仍计入（捕获也是活动）；删除的对象不计（以当前 vault 为准回填）。

### M6 活跃 Streak
- 定义：连续 `dayScore > 0` 的天数；端点规则见通用约定 4。
- 展示：当前 streak + 历史最长 streak。
- 边界：补写日记按其 `date` 计入（通用约定 5），允许"事后补救"，但当天无其他任何活动时单靠补记历史日期的日记不续今天。

## 3. 管道与质量

### M7 输入管道漏斗
- 各级计数：captured → extracted → summarized → reviewed → linked（archived 归入"已完结"合并展示；ignored 单列，不进漏斗）。
- **积压** = captured + extracted + summarized（存活口径）。
- **整体转化率** = (reviewed + linked + archived) / (全部 source − ignored)。
- **各级转化率** = 相邻两级计数之比，分母为 0 时显示 `—`。
- 边界：`11_原材料` 下无 frontmatter 的文件提示补全，不进统计。

### M8 知识成熟度分数
- 公式：`score = Σ concept(draft=1, verified=2, mature=3) + Σ method(candidate=1, usable=2, trusted=3, deprecated=0) + Σ research(draft=1, reviewed=2, complete=3, archived=0)`。
- 设计意图：不归一化、不设上限，分数随积累与审核单调上涨，服务成就感。
- 展示：总分 + 三类构成堆叠条 + "距下一整数里程碑还差 N 分"。

### M9 待审核中心计数
- 范围：`summary.reviewed=false` + `extract.review_status=pending` + `research.status=draft` + `concept.status=draft` + `reflection.status=raw` + `method.status=candidate` + `personal_operating_profile.status=draft`。
- 展示：状态栏常驻数字；清零时触发"审核清零"正反馈（见 M13）。

### M10 项目推进度与停滞
- 单项目推进度 = done 任务 / 全部任务（`31_任务` 中 `project` 指向该项目的 task + 项目页"当前任务" checkbox），无任务时显示 `—`。
- 停滞判定：`status=active` 且 `今天 − updated ≥ staleThresholdDays`（默认 3，可配）。
- 边界：`updated` 缺失时不判定，提示用户补全字段（通用约定 2）。

### M11 心情 / 精力趋势
- 数据：日记 frontmatter `energy`（整数）按周取均值画折线；`mood` 为自由字符串，不做数值化，只展示词频。
- 叠加：与 M5 的周日均 dayScore 放同一坐标，呈现状态与产出的相关性。
- 边界：`energy` 缺失的日记跳过；非整数按字符串忽略并提示。

### M12 系统健康分
- 公式：100 起扣，每个 validator error 扣 5 分、warning 扣 2 分，下限 0；6 个 validator（路径/Schema/状态/权限/Skill/Skill Eval）结果合并。
- 数据源：桌面端通过 kos-agent RPC `validate` 获取结构化 Validator 结果；不再调用 Vault Python Harness。移动端无本地 kos-agent 时显示“桌面端可用”。
- 展示：分数 + 扣分项列表，点击跳转问题文件。

## 4. 徽章与周期报告

### M13 徽章清单

| id | 名称 | 解锁条件（口径） | 可重复 |
|----|------|----------------|--------|
| first-concept | 首个概念 | concept 存活数 ≥ 1 | 否 |
| concept-50 | 概念半百 | concept 存活数 ≥ 50 | 否 |
| concept-100 | 概念百计 | concept 存活数 ≥ 100 | 否 |
| first-mature | 瓜熟蒂落 | mature concept ≥ 1 | 否 |
| summary-100 | 百篇摘要 | 累计创建 summary ≥ 100（累计口径） | 否 |
| streak-7 / streak-30 / streak-100 | 连续活跃 7/30/100 天 | M6 当前 streak 达标 | 否 |
| diary-30 | 笔耕不辍 | 累计日记 ≥ 30 篇 | 否 |
| task-100 | 任务百斩 | 累计完成任务 ≥ 100（completed 字段计数） | 否 |
| method-trusted | 千锤百炼 | trusted method ≥ 1 | 否 |
| method-validated-10 | 实践出真知 | Σ validated_times ≥ 10 | 否 |
| project-complete | 善始善终 | completed 项目 ≥ 1 | 否 |
| inbox-zero | 收件箱清零 | `10_收件箱` 文件数 = 0（首次及之后每次从非零回到零） | 是，记录次数 |
| review-clear | 审核清零 | M9 计数从非零回到 0 | 是，记录次数 |

- 解锁时弹动画并写入 data.json `badges`；可重复徽章记录 `inboxZeroCount` 等计数。

### M14 周报 / 月报
- 环比字段表：M2 各类型新增、M4 任务完成总数、M7 转化率变化与积压变化、M8 分数增量、M9 清零次数、M6 streak 维持情况、M13 新解锁徽章。
- 数据源：本周内每日快照 vs 上周同位置快照；快照缺失（插件未运行）按通用约定 6 处理并标注。
- 输出：一键生成 markdown，写入 `23_日记/YYYY/MM/` 对应周末日记的指定章节，或导出到剪贴板。

## 5. 口径冲突时的裁决

- 文档之间冲突：以本文档为准。
- 本文档与对象规范冲突：以对象规范为准，并回头修订本文档。
- 新增指标：先在本文档编号并写明公式与边界，再进 `metrics.ts`。
