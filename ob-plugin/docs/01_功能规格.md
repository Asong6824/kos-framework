# kos Obsidian 插件功能规格

> 版本：v0.1 草案
> 契约依据：`vault/90_系统/规则/对象规范.md`（对象类型、frontmatter、状态机、权限表）

## 1. 背景与定位

kos 现有的 `90_系统/harness/generate_daily_dashboard.py` 生成的是**静态、需手动触发的 Markdown 快照**。Obsidian 插件提供的是**实时、原生、可视化的交互层**：

- 直接读 Obsidian metadataCache（frontmatter 已被索引，无需自行解析），随编辑即时更新。
- 用原生视图、图表、按钮承载统计与操作，不污染 vault 内容。
- 与 harness 脚本互补，不替代：插件是"看板与入口"，harness 是"确定性校验器"。

边界原则：

- 插件遵守对象规范的路径、状态机与权限表。用户在插件中点击的状态流转按钮本质是"人的操作"，天然符合权限模型。
- 插件只写 frontmatter 状态位和模板化新文件，绝不修改正文人工撰写内容；`<!-- 人手动添加 -->` 块逻辑与现有 dashboard 脚本保持一致。
- 核心功能纯 TypeScript 实现（移动端可用）；调用 Python harness 仅作桌面端可选增强。

## 2. 可量化数据源盘点

对象规范中可用于量化展示的字段：

| 数据 | 来源 | 成就感叙事 |
|------|------|-----------|
| 概念数 + draft/verified/mature 分布 | `22_知识库` | "第二大脑有 N 个概念，M 个已成熟" |
| 输入源管道各状态计数 | `11_原材料` status | 输入→知识转化率、积压量 |
| 方法数 + validated_times | `40_方法库` | "方法被实践验证了 N 次" |
| 日记日期连续性 + mood/energy | `23_日记` | 连续记录 streak、精力趋势 |
| 任务 status + completed 日期 | `31_任务` | 今日/本周完成数、完成率 |
| 项目 updated + 页内任务进度 | `30_项目` | 项目推进度、停滞预警 |
| 所有对象的 created 字段 | 全部对象 | 知识资产复利增长曲线 |
| reviewed=false / draft 待审项 | 摘要/研究/概念/反思/方法 | 待办清零的掌控感 |
| 6 个 validator 结果 | harness | 系统健康分 |
| Signal importance | `50_信息雷达` | 雷达捕获的重要信号数 |
| 心情/精力（mood、energy） | 日记 frontmatter | 状态与产出的相关性 |

## 3. 功能清单

### A. 量化成就展示（核心重点）

**A1. 总览驾驶舱**（独立 tab 页）
- 知识资产总计：概念 / 方法 / 研究 / 项目 / 已处理输入源。
- 本周、本月新增 sparkline。
- 今日进度环：今日任务完成率、今日输入处理进度。

**A2. 活动热力图 + Streak**（GitHub 风格）
- 按天统计：创建对象数 + 完成任务数 + 是否写日记。
- 展示当前连续天数与历史最长记录。
- 历史数据靠对象 `created`/`completed` 字段回填，安装第一天即可画出完整热力图。

**A3. 知识复利曲线**
- 按 `created` 累计全部知识资产的增长曲线。
- 周环比标注："本周新增 12 个概念，比上周 +50%"。

**A4. 输入管道漏斗**
- captured → extracted → summarized → reviewed → linked 各级计数与转化率。
- 积压预警；Inbox Zero（`10_收件箱` 清空）达成时给正反馈。

**A5. 知识成熟度分布**
- 概念 draft/verified/mature、方法 candidate/usable/trusted 堆叠条。
- 汇总为单一"知识成熟度分数"，随审核操作上涨。

**A6. 成就徽章系统**
- 里程碑：首个 mature 概念、第 100 篇摘要、连续 30 天日记、方法晋升 trusted、Inbox Zero、周任务全清。
- 徽章墙 + 解锁动画。

**A7. 周报 / 月报自动生成**
- 对比上一周期的增量统计，一键写入 `23_日记` 或导出。

**A8. 心情 / 精力趋势**
- 日记 mood/energy 折线，与产出量叠加，呈现"状态与产出"的关系。

### B. 日常操作效率

**B1. 快速捕获**：全局快捷键弹窗，按模板落入 `10_收件箱/`，支持 URL/剪贴板预填。

**B2. 日课命令**：start-my-day / end-my-day（生成或打开今日工作台与日记）、一键刷新工作台。

**B3. 待审核中心**：汇总所有 `reviewed: false` 与 draft 状态的 AI 产物，逐项或批量 approve（一键改 frontmatter）。这是权限表"需人确认"环节的原生 UI。

**B4. 状态流转按钮**：笔记头部显示类型与当前状态徽章，按状态机给出合法的下一状态按钮，改动即时反映到所有统计。

**B5. 创建向导**：新建项目/概念/方法/任务，按模板生成路径与 frontmatter（等价 harness `create_*.py` 的纯 TS 实现，移动端可用）。

**B6. 聚合任务视图**：跨 `31_任务` 与项目页内 checkbox 的统一任务列表，按优先级/截止日排序，支持点击流转。

### C. 系统健康与治理

**C1. 健康分**：运行 6 个 validator（纯 TS 重实现核心规则，或调 Python），折算 100 分制，扣分项可点击跳转问题文件。

**C2. 停滞预警**：active 项目 ≥3 天未更新、captured 源积压超阈值时，状态栏黄点提醒。

**C3. Skill 生态统计**：core/incubator/personal 数量与 eval 覆盖率（面向框架维护者）。

### D. Harness 桥接（桌面端可选增强）

**D1.** 通过 child_process 调用 `90_系统/harness/*.py`（健康报告、每日简报），结果原生渲染。**核心功能不得依赖 Python**，否则移动端失效。

## 4. MVP 优先级

- **P0**：扫描索引器（metadataCache 驱动）+ A1 总览驾驶舱 + A2 热力图/streak + B1 快速捕获 + B3 待审核中心。形成"每天用、每天有反馈"的闭环。
- **P1**：A4 漏斗 + A5 成熟度分布 + B6 聚合任务视图 + B4 状态流转按钮 + C2 停滞预警。
- **P2**：A6 徽章 + A7 周报月报 + C1 健康分 + D1 harness 桥接 + A8 心情趋势。

## 5. 关键技术决策

1. **索引**：基于 Obsidian metadataCache 读取 frontmatter，事件驱动增量更新，不全量扫盘。
2. **历史回填**：streak、热力图、复利曲线的历史由对象 `created`/`completed` 字段回填；每日总数快照存插件 `data.json`，从安装日起累积。
3. **写入边界**：仅写 frontmatter 状态位与模板化新文件；状态流转严格按对象规范的状态机过滤可选项；权限表中"需人确认"的流转必须显式点击，不做自动化。
4. **图表**：轻量自绘 SVG，控制 bundle 体积。
5. **i18n**：中文优先，文案结构与 vault 文档术语对齐（如"摘录/摘要/认知记录"）。

## 6. 与 framework 的隔离

- 本插件源码与文档只放在仓库根目录 `ob-plugin/` 下，不进入 `vault/` 与 `dev/`。
- 插件对 kos 的唯一契约是 `vault/90_系统/规则/对象规范.md`；规范变更时需同步检查插件解析与状态机逻辑。
